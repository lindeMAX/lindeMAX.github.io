Charge Measurement on upcoming Muon Detectors - Scematics and electrical Simulation

Dec 9, 2021
3 min read
Charge Measurement Detector Scematics Simulation

I chose a voltage readout approach since i’ve already made some experiences with it.

This means that the SiPM’s photcurrent is driven through a resistor, which’s voltage drop is amplified by an voltage amplifier. After amplification there sits an integrator next to a comparator. The comparator will give us a short pulse, which is used to create timestamps for the event. The integrator will output a voltage, that is somehow related to the particles energy.

There is one major requirement the detector has to fulfill: Portability.
That means that it should be small in size and light weight, which will give us some restrictions:

  • Driven by a single-board-computer
    • USB power supply (5V DC)
  • Slow readouts
    • We need some kind of peak detection circuit or at least something that can hold the measured voltages/charges for a long enough time period
    • Pulse for timestamp needs to be long enough to be detected

Circuit Description

This is the circuit i came up with in the end.

It looks a bit overwhelming at first sight, i know. So let’s go into detail a little bit.

On the lower left there are some sources defined:

  • V2 - The main voltage source (5V)
  • V3 - A periodic square wave (Used to clear the charge measurement output)
  • B1 - An arbitrary behavioral current source, that’s used as a reference to the integration ciruit. It’s integrating the photocurrent.

On top of them is an exponetial current source, that’s used to simulate a current pulse from the SiPM, attached to a transmission line ($Z_0=50\Omega$).

The current pulses are driven through a $49.9\Omega$ input resistor and AC-coupled in the a non-inverting voltage Amplifier.

The amplified signal is then split up. One path goes to a Schmitt-Trigger and the other one to the integrator.
The rising edge of the trigger pulse shel then be used to generate timestamps.

The integrator has an additional non-inverting voltage amplifier attached to ist. This has two advantages:

  • The charge stored in $Cf$ can be hold over a longer time period
  • The outputvoltage, that shell be read by an ADC, can be tuned to fit into a specific interval

One may notice the mosfets attached to the ‘clear’ input. They have the prupose to discharge $Cf$ after a successful readout.

Simulation Results

V(v_amp) is the amplified pulse, V(adc) the output voltage of the integrator and V(clear) the signal that’s used to discharge the integrators capacitors.

v_amp is the peak value of the amplified pulse, v_charge the output voltage of the integrator and idt the integration over the SiPM’s current pulse performed by B1.
It is important to mention here, that the only modulated parameter of the SiPM’s current pulse is it’s height so far. That’s because the pulse width is nearly not varying in reality, too, and the real import thing is, that there is a linear relation between v_charge and idt. And this can be shown without varying height and width.
